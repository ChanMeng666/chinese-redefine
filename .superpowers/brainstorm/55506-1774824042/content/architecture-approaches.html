架构方案选择

Neon Database + Drizzle ORM + Better Auth — 部署平台决定了整体架构复杂度

A

✦ 推荐：完全迁移到 Vercel

移除 Cloudflare Workers，使用 Vercel 原生 Next.js 部署

优势

  • Better Auth + Drizzle + Neon 兼容性最佳，文档齐全
  • 原生 ISR + revalidatePath — 画廊缓存零配置
  • 迁移成本极低（仅删除 wrangler.toml 等 3 个文件）
  • 项目已有 .vercel 目录，可直接部署

劣势

  • Vercel 免费额度有限（100GB 带宽/月）
  • Serverless Functions 是区域性的（非全球边缘）
B

留在 Cloudflare Workers

继续使用 @opennextjs/cloudflare，用 KV 做缓存层

优势

  • Cloudflare 免费额度慷慨，全球边缘网络
  • 无需迁移，保持现有部署

劣势

  • Better Auth 在 Workers 中需要额外 polyfill
  • ISR 需手动实现（当前配置为 dummy）
  • @opennextjs/cloudflare 成熟度不如 Vercel 原生支持
  • 调试困难，兼容性风险高
C

混合方案：Vercel + Cloudflare CDN

Vercel 跑应用，Cloudflare 做 CDN/代理和边缘限流

优势

  • 兼得两者优势：Vercel DX + Cloudflare 安全防护
  • 边缘限流防 DDoS

劣势

  • 两个平台需要管理，配置复杂
  • 缓存一致性问题（两层缓存）
  • 对当前规模来说过度设计
