用户使用限制计划

三层防护体系 — 防滥用 + 控成本 + 预留商业化扩展

🛡️ 第一层：用户级别配额（核心）
限制维度 免费用户 未来付费用户
⏱️ 短期限流（防爆刷） 3 次 / 分钟 10 次 / 分钟
📅 每日配额 20 次 / 天 100 次 / 天
📆 每月配额 200 次 / 月 2000 次 / 月
💡 计算依据： $10-20/月 ≈ 3300-6600 次调用。若 30 个活跃用户 × 200 次/月 = 6000 次，刚好在预算内。日限 20 次防止单用户消耗过多月配额。
🚨 第二层：全局熔断器（紧急刹车）
全局日调用上限
500 次/天
达到后所有用户暂停生成
全局月调用上限
5000 次/月
≈ $15，留 buffer 不超预算
⚠️ 熔断逻辑： 用数据库中的 global_usage 表记录当日/当月总调用量。每次请求前检查，超限返回 503 + 友好提示。每日 00:00 UTC 重置日计数，每月 1 日重置月计数。
🔒 第三层：异常行为检测（反作弊）
🔄
重复词检测
同一用户 24h 内对同一词语重复生成，直接返回缓存结果（不消耗 API 调用）
🤖
自动化请求识别
短时间内连续请求不同词语（如 10 秒内 3+ 次），触发临时冷却 5 分钟
🚫
多账号检测（简易版）
同一 IP 注册超过 3 个账号时，新账号需人工审核（标记 flag 字段）
数据库设计
// 用户配额表 — 记录每个用户的使用量
┌─────────────────────────┐
│ user_usage               │
├─────────────────────────┤
│ id            SERIAL PK │
│ userId        TEXT FK    │  → users.id
│ date          DATE       │  → 哪一天（YYYY-MM-DD）
│ dailyCount    INT = 0    │  → 当日已用次数
│ UNIQUE(userId, date)     │
└─────────────────────────┘

// 全局用量表 — 熔断器依据
┌─────────────────────────┐
│ global_usage             │
├─────────────────────────┤
│ id            SERIAL PK │
│ date          DATE UNIQUE│  → 哪一天
│ dailyCount    INT = 0    │  → 当日全局调用量
│ monthlyCount  INT = 0    │  → 当月全局累计量
└─────────────────────────┘

// users 表新增字段
┌─────────────────────────┐
│ users (扩展)              │
├─────────────────────────┤
│ ...原有字段...            │
│ tier       TEXT = 'free' │  → 'free' | 'pro' (预留)
│ isFlagged  BOOL = false  │  → 异常标记
│ signupIp   TEXT          │  → 注册时 IP
└─────────────────────────┘
用户体验设计
✅ 正常状态
进度条显示：今日已用 3/20 次 · 本月 45/200 次
⚠️ 接近限额
警告提示：今日剩余 2 次生成机会
🚫 达到限额
生成按钮禁用 + 提示：今日配额已用完，明天再来吧！
🔥 全局熔断
全站提示：今日生成次数已达上限，服务将于明天恢复
